年轻人应该如何制作自己的动态库和静态库呢？让我们一起来看看吧！

下面有一些文件：

头文件mylib.h

#ifndef _MYLIB_H
#define _MYLIB_H

int func(int a);

#endif // !_MYLIB_H

源文件mylib.cpp

#include "mylib.h"

int func(int a)
{
    return a;
}

使用库的文件test.cpp

#include <iostream>
#include "mylib.h" // 需要函数声明，头文件提供

int main()
{
    std::cout << func(0) << std::endl;
    return 0;
}

使用Visual C++

制作静态库

文章假设你已经配置好Visual C++的环境

用Visual C++生成静态库的拓展名为.lib。就拿mylib.cpp做成静态库为例

首先要先生成目标文件，命令行输入 cl mylib.cpp /c
这样会生成mylib.obj目标文件
然后使用lib.exe生成静态库，输入 lib mylib.obj即可生成静态库文件mylib.lib

制作动态库

我们可以直接使用指令 cl mylib.cpp /LD 就可以生成动态库了

但是在生成动态库之前需要将需要的东西导出，要不然这个库是用不了的。我们只加上 __declspec(dllexport) 就可以导出了
__declspec是微软专有的关键字，不在标准里面。我们需要修改源文件，改成：

#include "mylib.h"

__declspec(dllexport) int func(int a)
{
    return a;
}

这样，在进行动态库生成，就可以了。此次生成了动态库文件mylib.dll，以及导入库mylib.lib。导入库是用来提供导出函数信息的，动态库提供机器指令。而静态库整个文件都是机器指令

此外，对于Visual C++生成的动态库.dll文件，还有一个可选的DLLMain函数，DLLMain函数是动态库的入口，就像main函数是程序的入口一样 DLLMain函数长这样： 

BOOL APIENTRY DllMain(
    HANDLE hModule,           // 指向自身的句柄
    DWORD ul_reason_for_call, // 调用库的原因
    LPVOID lpReserved )       // 隐式加载或显示加载
{
    switch (ul_reason_for_call)
    {
        case DLL_PROCESS_ATTACHED: // 被进程加载时
            break;
        case DLL_THREAD_ATTACHED: // 被进程释放时
            break;
        case DLL_THREAD_DETACH: // 被线程加载
            break;
        case DLL_PROCESS_DETACH: // 被线程释放
            break;
    }
    return TRUE;
}

当入口点函数返回 FALSE(定义为0) 值时，如果你使用的是加载时动态链接，则应用程序不会启动。 如果你使用的是运行时动态链接，则只有个别 DLL 不会加载；返回TRUE(定义为1)则会被加载

如何使用库

在源文件添加一句 #program comment(lib, "mylib.lib")即可，或者使用命令行 cl test.cpp /link mylib.lib
/link 是链接指令

无论动态库，静态库的使用方式都相同哦！

使用gcc

文章假设你已经配置好gcc的相关环境

生成静态库

gcc生成的静态库拓展名为.a，库名格式为 lib库名.a

首先使用命令行将源文件编译成目标文件，输入： gcc mylib.cpp -o mylib
这样会生成mylib.o目标文件。然后使用ar指令生成动态库
ar crv libmylib.a mylib.o
这样就生成了静态库libmylib.a文件

生成动态库

gcc生成的动态库拓展名为.so，格式为lib库名.so

首先使用命令行将源文件编译成目标文件， gcc mylib.cpp -o mylib
然后使用 -shared 指令生成动态库， -fPIC指令用于生成位置无关代码
（眩晕时间）位置无关代码，这样的代码本身就能被放到线性地址空间的任意位置，无需修改就能正确执行，而不生成位置无关的，加载时地址需要重定向。
此外，如果不生产位置无关的代码，源文件有一定的要求。在这里就不细说了。
例如 ：gcc -shared -fPIC libmylib.so -o mylib

这样就生成好了

使用库

在gcc，需要使用 -L. 指令来链接动态库或静态库（注意这里有一个点)，方法都是一样的，例如
gcc test.cpp -L. libmylib.a
gcc test.cpp -L. libmylib.so

以上就是制作动态库和静态库的相关内容，感谢您的阅读！
